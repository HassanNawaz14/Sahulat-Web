Section 1 — Sector Reality Check
⚡ Electricity (LESCO, GEPCO, FESCO, MEPCO, IESCO, PESCO, QESCO, HESCO, SEPCO, K-Electric) Richest Sector
Bill fetch: Every DISCO (distribution company) has a public bill check URL — enter reference number, get bill data. Fully scrapeable. LESCO: lesco.com.pk, K-Electric: ke.com.pk etc. No login needed.
Load shedding schedule: WAPDA/DISCOs publish weekly feeder-wise PDFs every Monday. Python + pdfplumber can parse these automatically. LESCO's PDFs are the most consistent.
Tariff slabs (NEPRA): Fully public at nepra.org.pk. 0–100 units at Rs. 7.74/unit, slabs go up to 700+ at Rs. 26.84/unit. Bill estimator is 100% buildable.
Slab boundary alert: Killer feature — tell the user "you've used 290 units, 10 more units will push you into the next slab and cost Rs. 800 extra." No app in Pakistan does this.
Complaint submission (NEPRA): NEPRA has a complaint web form at complaints.nepra.org.pk. No official API, but you can auto-fill and submit it using stored user data (Playwright/Puppeteer). Fragile but workable.
Real-time unscheduled outage data: No official source exists. Must be crowd-sourced — users report, system maps it. This is actually your biggest differentiator once you have users.
Bill payment: Requires JazzCash/EasyPaisa merchant account (needs business registration). V1 workaround: deep-link into JazzCash app with pre-filled consumer number. User pays in JazzCash, you just saved them the typing.
🔥 Gas (SNGPL — North, SSGC — South/Sindh) Moderate
Bill fetch: SNGPL (sngpl.com.pk) and SSGC (ssgc.com.pk) both have public bill lookup by consumer number. Fully scrapeable. Bill history available too.
Tariff slabs (OGRA): Published quarterly at ogra.org.pk. Residential slabs in MMBtu. Bill estimator buildable — just needs manual tariff updates quarterly.
Load management schedule: SNGPL publishes gas reduction schedules in winters on their website and Facebook, but not in a machine-readable format. Scrape their news/announcements section + crowd-source for real-time pressure issues.
Real-time pressure data: Doesn't exist publicly. Pure crowd-source only — "gas pressure low in my area right now."
Complaint submission: Both utilities have web complaint forms. Auto-fillable with stored user data. No API.
💧 Water (WASA Lahore, KW&SB Karachi, WASA Rawalpindi, others) Limited Data
Bill fetch: WASA Lahore (wasa.punjab.gov.pk) and KW&SB (kwsb.org.pk) have online bill check. Scrapeable. Many smaller cities (Multan, Faisalabad WASA) also have portals.
Supply schedule: WASA publishes supply timings in newspapers and Facebook occasionally — no consistent machine-readable source. Must be manually entered by admin or crowd-sourced by users.
Consumption tracking: Most homes have no smart meter. User manually enters meter reading. Your app calculates gallons used, projects monthly bill. Simple but valuable.
Water quality data: Does not exist publicly in Pakistan. Leave this for a future version.
Tanker tracking: Fully informal market. Could add "request water tanker" feature that links to local tanker services in major cities — affiliate-style.
🌐 Internet (PTCL, Nayatel, StormFiber, Jazz, Zong, Telenor) Moderate
Bill fetch: PTCL (ptcl.com.pk) and Nayatel have public bill check. StormFiber and others have customer portals. Scrapeable with login (user provides credentials, stored securely).
Package comparison: No site cleanly compares PTCL vs Nayatel vs StormFiber vs Jazz Home packages by area. You can build this as a static but regularly-updated module. High search traffic.
Outage tracking: ISPs don't publish outage info. Downdetector.pk exists but is thin. Crowd-sourced "my internet is down" reports aggregated by ISP + city is genuinely useful.
Usage tracking: Mobile data (Jazz/Zong/Telenor) has no public API. PTCL usage sometimes visible on portal. Mostly manual entry or user-provided credentials.
Complaint: PTA complaint portal (complaints.pta.gov.pk) accepts telecom complaints. Auto-fillable.
📺 Cable TV (Local operators, StormFiber TV, PTCL Smart TV) Weakest Sector
Reality: 90% of cable TV in Pakistan runs through informal local operators with no digital presence — monthly Rs. 500 collected door-to-door. No bill portals, no APIs, no data.
What's buildable: Manual expense tracker only — user logs "paid Rs. 700 to cable guy on 5th." PTCL Smart TV and StormFiber TV have portals but very small user base.
Recommendation: Include cable TV as a manual expense entry in the budget module only. Don't build a dedicated cable module — there's no data to work with.
☀️ Solar & Net Metering High Value, Underbuilt
Market timing: Pakistan installed 17,000+ MW of solar in homes in 2023–24 alone due to extreme electricity prices. This is the fastest-growing domestic utility in the country right now.
Inverter APIs: Growatt (most common brand in Pakistan) has a free ShineMonitor API. Solis has SolisCloud API. Huawei FusionSolar has API access. User connects their inverter account — you pull daily production data automatically.
Net metering calculator: NEPRA's net metering policy (Rs. 27/unit exported) is public. Calculate how much the user is saving and how much DISCO owes them this month.
ROI tracker: User enters system cost (e.g. Rs. 1,200,000 for 10kW), you track monthly savings and tell them "your system will pay back in 3.2 years at current rates."
Panel health alerts: If production drops unexpectedly vs sunny-day baseline, alert user that a panel may be dirty or underperforming.
Solar sizing tool: User enters monthly electricity units → app recommends system size (kW) and estimated cost. High affiliate potential with solar companies.
Section 2 — Module Architecture

Core Modules (Build in V1)

M1 — Bill Tracker

Fetch + store bills by consumer number for electricity, gas, water, internet. Show monthly trends. No login needed for most utilities.

M2 — Consumption Monitor

Manual meter reading entry with daily/monthly unit tracking. Slab boundary alerts. Projected end-of-month bill before the actual bill arrives.

M3 — Outage Tracker

Electricity load shedding (PDF-parsed, scheduled). Crowd-sourced real-time reports for gas, water, internet. Push notifications before scheduled cuts.

M4 — Bill Estimator

Enter units → get exact estimated bill using NEPRA/OGRA/WASA tariff slabs including taxes, surcharges, and meter rent.

M5 — Budget Manager

Monthly utility spending dashboard. Set budget per utility. Track actuals. Add other expenses (education, grocery). Monthly summary report.

M6 — Solar Dashboard

Connect Growatt/Solis/Huawei inverter. See daily production vs consumption. Net metering savings. ROI countdown. Panel health alerts.

Secondary Modules (Build in V2 — post first users)

M7 — Complaint Assistant

User's details stored once. App auto-fills NEPRA, OGRA, PTA, WASA complaint forms. Tracks complaint reference numbers and status.

M8 — Pay Gateway (Lite)

Deep-link to JazzCash/EasyPaisa with pre-filled consumer number + amount. Not actual payment processing — just eliminates manual entry friction.

M9 — ISP Comparison

Compare internet packages by city/area. PTCL vs Nayatel vs StormFiber speeds, prices, reliability scores (crowd-sourced). High affiliate potential.

M10 — Community Reports

Hyperlocal feed: "Gas low pressure in DHA Phase 5 right now — 14 reports." Outage map. Verified vs unverified reports. WhatsApp share button.

M11 — Solar Sizing Tool

Enter avg monthly units → get recommended system size, estimated cost, and payback period. Affiliate links to vetted solar vendors.

M12 — Usage Insights

Data science layer: seasonal patterns, unusual consumption spikes, appliance cost breakdown estimator ("your AC costs ~Rs. 4,200/month to run").

Section 4 — Monetization Strategy
Google AdSense: Primary revenue from day one. High daily session count (users check 3–5x per day) means high impressions. Solar and UPS brands have high CPCs in Pakistan.
Daraz Affiliate: UPS, inverters, batteries, solar panels, energy-saving appliances. A user who just got a Rs. 8,000 electricity bill is a very motivated buyer. Link them to relevant products.
Solar company affiliate: Solar installers in Pakistan charge Rs. 800,000–2,000,000 per system. If your tool recommends a system and the user contacts a vendor via your referral, charge the vendor Rs. 3,000–10,000 per lead. Very achievable once you have traffic.
ISP referral: Nayatel, StormFiber, and others pay referral commissions. Your ISP comparison module sends them verified, intent-heavy leads.
Premium tier (later): Rs. 99/month for SMS outage alerts (not everyone has data during outage), bill history export to PDF, multi-home tracking. Optional — ads alone will sustain early growth.

Section 6 — Summer Build Roadmap

Week 1–2 — Core Data Layer

Build scrapers for LESCO bill fetch + load shedding PDF parser. Build SNGPL + SSGC bill scraper. Set up Supabase schema: users, consumer_numbers, bills, outage_schedules, reports. Deploy scrapers on Railway cron.

Week 3 — Frontend MVP

Build Next.js PWA. Screens: Dashboard, Add Utility (enter consumer number), Bill View (fetched bill + trend graph), Outage Schedule (today/tomorrow + countdown timer). Deploy on Vercel. Make it a PWA so it can be installed.

Week 4 — Engagement Features

Add push notifications (15-min outage warning). Add slab boundary alerts ("you're 20 units from the next slab"). Add one-tap outage reporting. Add WhatsApp share button for schedules. Apply for Google AdSense.

Week 5 — Budget + Estimator

Build bill estimator (enter units → get exact bill). Build budget dashboard (monthly utility spend tracker). Add NEPRA + OGRA tariff data. Add basic expense tracker for other categories.

Week 6 — Solar Module + AdSense Live

Integrate Growatt API for solar users. Build net metering calculator. AdSense should be approved by now — first revenue. Add Daraz affiliate links to UPS/battery product recommendations.

Post-Summer — Scale

Add remaining DISCOs (GEPCO, FESCO, MEPCO etc). Add WASA water bill. Add ISP comparison module. Add complaint auto-fill. Onboard solar vendor affiliates. Each addition = new user wave.

Section 7 — Project Name
Sahulat Urdu — "ease / convenience". Every Pakistani knows this word. Domain: sahulat.pk or sahulat.app

Alternative names: BillSaathi (bill companion), GharHisaab (home accounts), UtiliPK (utilitarian, memorable). Sahulat is the strongest — it speaks to the promise of the app in one word that resonates across all demographics.

⚠️ Honest caveat: Cable TV and mobile data tracking are the two weakest modules — no public data exists for either. Build them as manual expense trackers only and don't over-promise. Everything else in this draft is genuinely buildable, for free, with your current skill set.
